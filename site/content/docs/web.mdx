Earl ships with a built-in web playground — a local web UI for browsing, testing, and debugging your templates without touching the command line.

```bash
earl web
```

Earl starts a local server, prints its URL and a bearer token, and opens your browser automatically.

```
earl web playground listening at http://127.0.0.1:52413
Authorization: Bearer KT_J0q0KH-qP0FbYlgwIPk06jceacaRViPxEQYFWlzE
```

The web UI is compiled into the `earl` binary — no separate install, no Node.js runtime, no external dependencies.

---

## Overview

The playground has three panels:

![Web playground overview](/earl/images/web/web-overview.png)

- **Sidebar** (left) — All loaded templates grouped by provider, with a search filter at the top.
- **Documentation** (center) — Details for the selected command: description, parameters, required secrets, categories, and source file.
- **Playground** (right) — An interactive form for filling in parameters and executing commands.

---

## Browsing Commands

All templates from your local `./templates/` directory and global `~/.config/earl/templates/` directory are loaded automatically. The sidebar groups commands by provider and shows the count next to each.

Use the search box at the top of the sidebar to filter commands by name:

![Filtering commands](/earl/images/web/web-search.png)

Selecting a command shows its full documentation in the center panel, including:

- **Protocol and mode** — The protocol (HTTP, GraphQL, gRPC, Bash, SQL) and whether the command is read or write mode.
- **Parameters** — Each parameter with its type, whether it is required, its default value, and a description.
- **Secrets** — Which secrets the command needs, with status indicators showing whether each is configured.
- **Source** — The template file path.

---

## Executing Commands

The right panel provides an interactive form for the selected command. Parameters are rendered as appropriate input fields (text inputs, number spinners) with placeholder text from the parameter descriptions.

![Command with parameters](/earl/images/web/web-params.png)

Click **Execute** (or press `Cmd+Enter` / `Ctrl+Enter`) to run the command. Results appear below with three view modes:

- **Raw** — The raw response body.
- **Decoded** — Values extracted via any configured `json_pointer` or response processing.
- **Human** — A human-readable formatted output.

![Execution results](/earl/images/web/web-execute.png)

The response header shows the exit code (or HTTP status), latency, and response size.

<Callout type="info">
  Write-mode commands require an extra confirmation step in the web UI to prevent accidental mutations.
</Callout>

## Code Examples

Expand the **Code** section to see the equivalent CLI command for the current parameters. For HTTP commands, a **cURL** tab is also available.

![CLI code example](/earl/images/web/web-code.png)

This makes it easy to copy a working command into scripts or share it with teammates.

---

## Request History

The **History** drawer at the bottom of the playground tracks all commands executed during the session. Click any past request to review its parameters and response.

---

## CLI Options

| Flag              | Default       | Description                                             |
| ----------------- | ------------- | ------------------------------------------------------- |
| `--listen <addr>` | `127.0.0.1:0` | Bind address. Port `0` selects a random available port. |
| `--no-open`       | off           | Do not automatically open the browser.                  |

```bash
# Default: random port, opens browser
earl web

# Fixed port, no browser
earl web --listen 127.0.0.1:3000 --no-open
```

## Security

The web server is designed for local development use:

- **Bearer token** — A random token is generated on each startup and required for all API requests. The token is injected into the served page automatically.
- **Localhost only** — The default bind address is `127.0.0.1`, so the server is not accessible from other machines.
- **Session-scoped** — The token is only valid for the current session. Restarting the server generates a new token.
- **Write confirmation** — Write-mode commands require explicit confirmation in the UI, matching the `--yes` safety model used by the CLI.

Press `Ctrl+C` to stop the server.
